### Recommendations (prioritized, with effort estimate)

   - Replace bare `catch {}` with `catch (error) { console.warn('Credentials file unreadable:', error); return null }`.
   - Alternatively, create a `CredentialsError` class and re-throw so the CLI can provide a specific message.

2. **Log the JSON parse fallback in `packages/cli/src/lib/client.ts`** (P2, ~10 min)
   - In `parseErrorBody`, add `console.warn('Failed to parse API error body:', error)` inside the bare `catch` before returning the fallback.

3. **Create a `CliAuthError` class in `packages/cli/src/lib/client.ts`** (P2, ~15 min)
   - Replace `throw new Error('Not authenticated...')` with `throw new CliAuthError('...')` so callers can handle it specifically.

4. **Create a `ThemeError` class in `packages/ui/src/lib/theme.ts`** (P3, ~20 min)

5. **Introduce `ApiPathError` in `packages/cli/src/lib/client.ts`** (P3, ~5 min)
   - Replace the path validation `throw new Error` with a dedicated class.
