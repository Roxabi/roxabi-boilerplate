### Recommendations
1. **Replace empty catch with structured logging** (P1) — In `design-system/index.lazy.tsx`, replace `catch (e) {}` with a `console.warn` or Sentry capture so corrupted localStorage themes are visible. Effort: XS (~5 min).
2. **Introduce a `fetchError` helper** (P2) — Create a shared `fetchError(res, fallback)` that throws `new Error` with the status code AND response message (e.g., `[500] Failed to fetch users: {body}`). Apply to all 10 `throw new Error('Failed to fetch...')` call sites in admin routes. Effort: S (~30 min).
3. **Replace generic catch toasts with contextual messages** (P2) — In auth flows (`login-handlers.ts`, `register.tsx`, `verify-email.tsx`, `reset-password.tsx`, `account-reactivation.tsx`), capture the original error and include it in the toast or at least `console.error`. Effort: S (~45 min).
4. **Add logging to silent `.catch(() => {})` patterns** (P2) — In `dashboard.tsx` (authClient calls), `register.tsx` (consent sync), `magic-link/verify.tsx` (signOut), add `console.warn` or telemetry. Effort: XS (~15 min).
5. **Make `fetchRoles` fail visibly** (P2) — In `admin/members.tsx`, change `fetchRoles` to throw or return an error object instead of silently returning `[]`. Effort: XS (~10 min).
6. **Add route-level `errorComponent` to root** (P3) — `__root.tsx` has no `errorComponent`. While `beforeLoad` is safe today, adding one provides a fallback for future loader errors. Effort: XS (~10 min).
