# T1 — API Unit / Integration Test Quality Audit

## Summary

113 test files audited. One critical tautology test in `auth.integration.spec.ts` (line 47) can never fail because it computes dependency checks but always asserts `expect(true).toBe(true)`. Three integration spec files contain 34 `expect(true).toBe(true)` fallback paths that silently pass when the module fails to compile. Eight files use module-level `vi.mock` that hides real behavior. One tenant integration suite silently skips when `DATABASE_URL` is absent. Seventeen controller tests share a single instance across all test cases.

## Recommendations

| Priority | Finding | Action | Effort |
|----------|---------|--------|--------|
| 1 | P0 tautology in `auth.integration.spec.ts` | Replace `expect(true).toBe(true)` with actual assertions on `_hasValidateToken` and `_hasHashPassword`. | 5 min |
| 2 | 34 `expect(true).toBe(true)` across 3 integration spec files | Replace all fallback paths with `fail()` or `expect(module).toBeDefined()` so broken compilation fails the test instead of passing silently. | 30 min |
| 3 | 8 files with module-level `vi.mock` | Replace module-level `vi.mock` with inline `vi.spyOn` or factory mocks scoped to individual tests. Start with `auth/auth.instance.test.ts` (7 mocks). | 2-3 h |
| 4 | `tenant/tenant.integration.test.ts` | Replace `describe.skipIf` with `describe` and require `DATABASE_URL` in CI; or use `it.skipIf` on individual tests with a reason comment. | 15 min |
| 5 | 17 shared controller instances | Move controller instantiation inside `beforeEach` or individual test blocks to prevent accidental cross-test state leakage. | 1 h |
| 6 | `auth/auth.instance.test.ts` | Reset `capturedConfig` and `capturedMagicLinkConfig` in `afterEach` as well as `beforeEach` to ensure zero leakage. | 10 min |
