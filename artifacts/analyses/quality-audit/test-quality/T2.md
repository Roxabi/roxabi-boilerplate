### Summary
Web tests cover UI rendering and auth flows well, but have critical gaps in tenant isolation coverage, pervasive `globalThis.fetch` teardown omissions, and missing integration tests for route guards/filters. Several files share mutable captured state across tests without reset, creating flaky conditions.

### Recommendations
1. **Add tenant isolation tests (P0, effort: high)**
   - In `admin/members.test.tsx`, `admin/settings.test.tsx`, `dashboard.test.tsx`, and `apiKeys.test.ts`, add tests that simulate a user belonging to `org-1` attempting to access `org-2` data. Verify the UI/API calls reject or redirect appropriately.
   - For `apiKeys.test.ts`, verify that `listApiKeys` only returns keys scoped to the active organization.

2. **Add route guard integration tests (P1, effort: medium)**
   - For every authenticated route (`dashboard`, `admin/*`, `settings/*`), add a `beforeLoad` test that verifies `enforceRoutePermission` or `requireAuth` is called and redirects unauthenticated users.
   - For guest routes (`login`, `register`, `reset-password`), add `beforeLoad` tests that verify `requireGuest` redirects authenticated users.

3. **Fix `globalThis.fetch` teardown (P1, effort: low)**
   - Add `let originalFetch: typeof globalThis.fetch` and `beforeEach(() => { originalFetch = globalThis.fetch })` / `afterEach(() => { globalThis.fetch = originalFetch })` to `register.test.tsx`, `profile.test.tsx`, `admin/settings.test.tsx`, `CreateFlagDialog.test.tsx`, and `feature-flags.test.tsx`.

4. **Reset shared `captured` state between tests (P2, effort: low)**
   - Add `beforeEach(() => { captured.loaderData = createDefaultLoaderData() })` to `login.test.tsx` and `register.test.tsx` to prevent `loaderData` leak between tests.

5. **Replace `Object.defineProperty(window, 'location')` with `vi.stubGlobal` (P2, effort: low)**
   - In `magic-link/verify.test.tsx`, use `vi.stubGlobal('location', { ...window.location, reload: reloadMock })` and `vi.unstubAllGlobals()` in `afterEach`.

6. **Restore `document.cookie` in consent tests (P2, effort: low)**
   - In `consentProvider.test.tsx`, save the original `document.cookie` before the test and restore it in `afterEach`, or use a cookie mocking utility.

7. **Add API keys route page tests (P3, effort: medium)**
   - Create a `settings/api-keys.lazy.test.tsx` that tests the route component rendering, list loading, create dialog flow, and revoke confirmation flow.
