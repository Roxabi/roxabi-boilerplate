### Recommendations (prioritized, with effort estimate)
1. **Replace SSR localhost fallbacks with env-driven base URL** (P1, effort: 2h) — `authClient.ts` and `routePermissions.ts` should read `APP_URL` / `API_URL` from env or throw at build time instead of defaulting to localhost. In `authClient.ts`, pass `baseURL` from server-side config.
2. **Centralize role name constants** (P2, effort: 30m) — Extract `'member'`, `'superadmin'` into a shared `@repo/types` or `@repo/config` constant. The `superadmin` string is already defined in the backend; it should be shared.
3. **Extract shared `BAN_REASON_LIMITS` constant** (P2, effort: 20m) — Create a single `const BAN_REASON_LIMITS = { min: 5, max: 500 }` in `components/admin/constants.ts` and use it in both `BanDialog.tsx` and `UserActions.tsx`.
4. **Centralize query `staleTime` defaults** (P2, effort: 30m) — Define a `QUERY_STALE_TIME = 30_000` constant in `lib/queryConfig.ts` (or `@repo/config`) and reference it from all `useQuery` hooks.
5. **Fix GDPR export error handling** (P2, effort: 15m) — Show a `toast.error()` in the `catch` block of `PrivacyDataSection.tsx` instead of silently swallowing the failure.
6. **Fix approximate consent duration math** (P2, effort: 20m) — In `consentProvider.tsx`, replace `6 * 30 * 24 * 60 * 60 * 1000` with `Date` arithmetic (e.g., `new Date(Date.now() - 6 * 30 * 24 * 60 * 60 * 1000)` is still approximate; better to compute exactly using `addMonths` or `Date` object manipulation).
7. **Remove duplicate JSDoc in `UserActions.tsx`** (P2, effort: 5m) — Delete the second JSDoc block at lines 328-333.
8. **Extract `retryDelay` constant in `apiClient.server.ts`** (P3, effort: 10m) — Name the 500ms delay and make it configurable via env or at least a named constant at the top of the file.
9. **Remove hardcoded ports from `HeroSection.tsx`** (P3, effort: 10m) — Use a configurable terminal feed config or accept that this is demo text and add a comment noting it is illustrative only.
