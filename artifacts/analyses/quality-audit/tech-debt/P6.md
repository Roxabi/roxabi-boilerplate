### Recommendations
1. **Remove console.error calls** (P1, 3 issues) — Replace with `toast.error()` or silently swallow. The error messages are already shown to users via toast; console logging leaks implementation details. Effort: XS (~10 min).
2. **Replace window.location mutations with router navigation** (P1, 4 issues) — `magic-link/verify.tsx`, `admin/settings.tsx`, and `api-keys/error-state.tsx` all use `window.location.reload()` or `window.location.href` which bypass React Router and cause full page reloads. Use `navigate()` or query refetch instead. Effort: S (~30 min).
3. **Externalize legal config** (P1, 1 issue) — Move `legal.config.ts` values to environment variables or a config file that is clearly marked as requiring customization before deploy. The current placeholder data creates legal non-compliance risk. Effort: S (~1 h).
4. **Resolve adminMemberKeys TODOs** (P2, 4 issues) — Follow up on issue #410 to migrate the 4 inline query key arrays to the factory pattern. This is a consistency/correctness issue for cache invalidation. Effort: S (~45 min).
5. **Extract timer constants** (P2, 8 issues) — Define shared constants like `COOLDOWN_TICK_MS`, `COPY_FEEDBACK_MS`, `DICEBEAR_URL_MAX_LENGTH` in a central config file. The 1000 ms timers are copy-pasted across 4 files. Effort: S (~30 min).
6. **Address ComponentsSection TODO(#90)** (P2, 2 issues) — Type `ComponentShowcase` children properly instead of using `Record<string, unknown>` and `as` assertions. Effort: M (~1.5 h).
7. **Clean up stale comments** (P2/P3, 2 issues) — Update `useCursorPagination.ts` comment to remove "Phase 2" reference; remove or repurpose `CHROMELESS_PREFIXES` in `__root.tsx`. Effort: XS (~15 min).
8. **Consolidate biome-ignore suppressions** (P3, 7 issues) — The `noArrayIndexKey` suppressions in skeleton lists are all legitimate. Consider extracting a shared `SkeletonList` component so the suppression is centralized to one location. Effort: S (~45 min).
