### Recommendations
1. **Extract admin detail page sub-components** (P1 x3, effort: M) — Split `admin/users.$userId.tsx`, `admin/organizations.$orgId.tsx`, and `admin/settings.tsx` into colocated `/-components/` directories (e.g., `ProfileCard`, `EditForm`, `DangerZoneCard`). Move shared hooks to `hooks/` or `lib/admin/hooks.ts`.
2. **Unify query key factories** (P1, effort: S) — Resolve the TODOs in `admin/members.tsx` by introducing `adminMemberKeys` and `adminInvitationKeys` factories, replacing hardcoded strings.
3. **Create shared admin UI primitives** (P2 x3, effort: S) — Move `EmptyState`, `TableSkeleton`, and `FilterBar` handlers into `components/admin/` or `@repo/ui` to eliminate duplication across 5-7 admin pages.
4. **Extract `useCooldown` hook** (P2, effort: XS) — Merge the duplicated `useResendCooldown` / `useResendCooldownEffect` logic into a single `useCooldown` hook in `hooks/` and replace usages in `verify-email.tsx`, `magic-link-sent.tsx`, `reset-password.tsx`, and `login.tsx`.
5. **Move `slugify` to shared lib** (P2, effort: XS) — Extract `slugify` to `lib/slugify.ts` (or `@repo/ui`) and unify the two implementations; ensure both callers use the same canonical version.
6. **Split long components** (P2 x5, effort: S) — Break `EditUserForm`, `AdminUsersList`, `AdminAuditLogsPage`, `AvatarCustomizationSection`, and `OrgResolutionStep` into smaller focused components (e.g., extract the table body or form fields into separate files).
7. **Remove biome-ignore suppressions** (P2, effort: XS) — Fix the missing dependency in `settings/api-keys/-hooks.ts` (line 13) instead of suppressing it; add the missing dependency to the `useEffect` array.
8. **Move domain logic out of UI routes** (P3, effort: XS) — Move `SoftDeletableOrg`, `hasSoftDeleteFields`, and `slugify` from `admin/settings.tsx` to `lib/admin/` or `@repo/types` to prevent domain logic leakage into the presentation layer.
