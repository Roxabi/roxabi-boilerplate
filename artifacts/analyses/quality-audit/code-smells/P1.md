### Summary
Partition P1 (AuthN/AuthZ, RBAC, API keys) contains 1 critical regex bug that rejects valid default permissions, several DRY violations (email templating, permission mapping), dead code, and an N+1 query in role seeding. Tenant isolation via RLS is correctly applied; no critical security gaps were found in the audit.

### Findings
| severity | file | line | description |

### Recommendations (prioritized, with effort estimate)

1. **Fix permission regex inconsistency (P0)** — 30 min
   - Unify `PERMISSION_FORMAT` and the API key regex into a single source of truth (e.g. `packages/types` or `@repo/config`). The regex should match the actual permission strings used: `^[a-z][a-z_-]*:[a-z][a-z_-]*$`. Update both `rbac.controller.ts` and `apiKey.controller.ts` to import it.

2. **Extract email helper in `auth.instance.ts` (P1)** — 45 min
   - Create a private `sendTemplatedEmail(params)` function that encapsulates: render with fallback, enqueue, log error. Replace the 4 duplicated blocks (~80 lines → ~20 lines).

3. **Hoist `allPerms` query outside loop in `seedDefaultRoles` (P1)** — 10 min
   - Move `qb.select().from(permissions)` before the `for` loop.

4. **Extract `resolvePermissionIds` helper (P2)** — 20 min
   - A 5-line helper that takes `permissions: string[]` and `allPerms` and returns the `inserts` array. Removes 3 occurrences.

5. **Extract `assertSlugAvailable` helper (P2)** — 15 min
   - Remove partial duplication between `createRole` and `updateRole` slug-collision checks.

6. **Remove dead exception classes (P2)** — 5 min
   - Delete `ApiKeyExpiredException` and `ApiKeyRevokedException` (or wire them into `validateBearerToken` if they should be used instead of `ApiKeyInvalidException`).

7. **Refactor `countMembersWithRole` to SQL `count()` (P2)** — 10 min
   - Replace `select({ id: members.id }).from(members)` with `count()` aggregate.

8. **Deduplicate `findRoleById` query (P2)** — 20 min
   - Either extract a shared `findRoleById` utility or consolidate `DrizzleRbacMemberRepository` role queries into `DrizzleRbacRepository` and inject it into `RbacMemberService`.

9. **Move `RbacListener` queries to repository (P2)** — 20 min
   - Add `findOwnerRole(tenantId)` and `assignOwnerRole(creatorUserId, tenantId)` methods to the repositories and use them in the listener.

10. **Refactor `AuthGuard` soft-delete route list (P2)** — 30 min
    - Use a route-registry or metadata-driven approach instead of hardcoded strings.

11. **Remove unused `hasPermission` or promote it (P3)** — 5 min
    - If not needed, delete it. If it is a planned feature, add a TODO or implement the call site.

12. **Remove redundant type assertions (P3)** — 5 min
    - Clean up `(user as UserWithLocale).locale` casts.
