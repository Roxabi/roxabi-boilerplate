### Recommendations (prioritized, with effort estimate)

1. **Wrap `batchUpdate` writes in a transaction** (P2, effort: S)
   - Pass `tx` through `SystemSettingsService.batchUpdate` to `DrizzleSystemSettingsRepository.updateByKey`, or collect all updates and run them inside a single `db.transaction` block. Prevents partial-batch state.

2. **Move `removeMember` last-owner check into transaction** (P2, effort: S)
   - Wrap `ensureNotLastOwner` and `delete` in the same `db.transaction` block, similar to `changeMemberRole` (line 157). Eliminates the TOCTOU race.

3. **Replace recursive N+1 org hierarchy queries with a single CTE** (P2, effort: M)
   - `getSubtreeDepth` and `getDescendantOrgIds` can be replaced by a single recursive PostgreSQL CTE (Drizzle supports `withRecursive`). Reduces arbitrary N queries to 1 query per call.

4. **Use `tx` consistently inside `validateHierarchy`** (P2, effort: XS)
   - Change `getSubtreeDepth(db, orgId)` to `getSubtreeDepth(tx, orgId)` (line 46) so both reads share the same transactional snapshot.

5. **Add cache stampede protection to feature flags** (P3, effort: S)
   - Add a `Promise` cache (pending fetches) in `FeatureFlagService.isEnabled` so concurrent requests for the same key share a single DB query instead of stampeding.

6. **Document fire-and-forget audit semantics** (P3, effort: XS)
   - Add a comment block or ADR noting that audit logs are intentionally fire-and-forget. Consider a background queue or outbox pattern if audit durability becomes a hard requirement.
