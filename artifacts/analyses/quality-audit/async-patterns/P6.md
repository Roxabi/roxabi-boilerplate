### Summary
No critical (P0/P1) async issues found in partition P6 (Web routes, integrations, hooks). Eleven medium (P2) issues were identified: floating promises in `useEffect` and callbacks, unhandled clipboard rejections, silent error swallowing in mutation wrappers, sequential API calls in a loop, and a missing individual catch on a parallel `beforeLoad` fetch.

### Recommendations
1. **Wrap `getServerEnrichedSession()` with `.catch(() => null)`** in `__root.tsx` so a session fetch failure does not crash the entire route load.
2. **Await `fetch('/api/consent')` in `register.tsx`** or track the promise with a ref so the handler does not resolve before the consent sync completes.
3. **Add `.catch(() => {})` to `navigator.clipboard.writeText`** in `CodeSnippet.tsx` (or switch to async/await) to prevent unhandled rejections.
4. **Remove `.catch(() => {})` from `feature-flags.tsx` callbacks** and let `useMutation` `onError` handle toasts. The current wrapper hides the error from the hook.
5. **Add `await` to `queryClient.invalidateQueries` calls** in `handleActionComplete` / `onActionComplete` callbacks, or return the promise so the caller can await it.
6. **Refactor `fetchOwnedOrgsForUser`** in `settings/-account-delete.tsx` to use `Promise.all` for parallel org fetching.
7. **Add a cleanup ref to `dashboard.tsx` `useAutoSelectOrg`** so the `setActive` promise is ignored if the component unmounts.
