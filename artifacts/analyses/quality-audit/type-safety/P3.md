### Recommendations (prioritized, with effort estimate)

1. **Replace error `as` casts with proper type guards** — `M` effort
   - Files: `adminOrganizations.service.ts`, `adminUsers.service.ts`, `adminInvitations.service.ts`, `adminFeatureFlags.controller.ts`, `logAudit.ts`
   - Introduce a shared `isPgError(err)` and `isError(err)` type guard in `apps/api/src/common/` and replace all `err as { code?: string }` and `err as Error` patterns. This is a cross-cutting fix that improves safety in all catch blocks.

2. **Replace `payload as Parameters<AuditService['log']>[0]` with proper typed construction** — `S` effort
   - Files: `adminInvitations.service.ts:258`, `adminMembers.service.ts:332`
   - Build the payload object inline using the `AuditService['log']` parameter type instead of casting an untyped `Record<string, unknown>`.

3. **Fix `row.value as T` in `SystemSettingsService.getValue`** — `S` effort
   - File: `systemSettings.service.ts:14`
   - Add a runtime type guard or JSON schema validation per setting type, or change the API to return `unknown` and let callers validate. The current generic cast is a type safety hole.

4. **Add explicit return types to exported standalone functions** — `XS` effort
   - Files: `adminUsers.shared.ts`, `adminOrganizations.shared.ts`, `admin/utils/logAudit.ts`
   - Four exported functions missing return types; adding them improves API surface clarity and catches drift during refactoring.

5. **Remove `biome-ignore` non-null assertions in `adminMembers.service.ts`** — `XS` effort
   - Lines: 288, 305
   - Replace `member.roleId!` with an explicit null check or narrow the type upstream so the assertion is unnecessary.

6. **Clean up test file `as unknown as` casts** — `XS` effort
   - Use `vi.mocked()` or proper mock factories with typed stubs instead of structural `as unknown as` casts. 22 occurrences across test files.
