### Recommendations
1. **Replace CLS `as string` casts with runtime validation** (P1, effort: low) — Add a small `getTenantId()` helper that validates `cls.get('tenantId')` is a string and throws a clear error otherwise. Apply in `rbac.service.ts` and `rbacMember.service.ts`.
2. **Add runtime role validation in `buildApiKeySession`** (P1, effort: low) — Validate `keyData.role` against the `Role` union before casting; throw if the DB contains an invalid role value.
3. **Remove `as` casts from `isAuthenticatedSession` type guard** (P2, effort: medium) — Replace `value as Record<string, unknown>` with `typeof` checks and `in` operator checks, or use `zod` for runtime validation.
4. **Fix `authorization` header type narrowing** (P2, effort: low) — Use `Array.isArray` check to handle `string[]` case before narrowing to `string`.
5. **Add explicit return types to public controller and service methods** (P3, effort: medium) — Prioritize `auth.controller.ts`, `rbac.controller.ts`, `apiKey.controller.ts`, and `auth.service.ts`/`rbac.service.ts`/`apiKey.service.ts` public methods.
6. **Remove non-null assertion in `generateBase62`** (P2, effort: low) — Use a type-safe index access or replace the loop with `Array.from` and a typed map.
7. **Add a biome rule or lint:custom check for `as` casts in auth/rbac/api-key paths** (P3, effort: low) — Prevent future regressions in the critical security partition.
