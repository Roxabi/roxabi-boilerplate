### Recommendations (prioritized, with effort estimate)

1. **Introduce API response Zod schemas for all `res.json()` casts** (P2, effort: medium, ~4h)
   - Follow the pattern already used in `featureFlags/queries.ts` (`z.array(featureFlagSchema).parse(await res.json())`).
   - Files to update: `apiKeys.ts`, `profile.ts`, `authClient.ts`, `featureFlags/api.ts`, `consent/consentProvider.tsx`, `components/admin/PendingInvitations.tsx`.
   - Add shared schemas in `@repo/types` or local `zod` schemas where appropriate.

2. **Replace `ctx: any` in `enforceRoutePermission` with a branded TanStack Router context type** (P2, effort: small, ~1h)
   - Use `RouteApiContext` or `BeforeLoadContext` from `@tanstack/react-router` if available; otherwise define a minimal `Pick<>` interface that covers `matches`, `routeId`, and `context.session`.
   - Remove the `biome-ignore` comment.

3. **Add runtime validation to `shiki.ts` before `as` casts** (P2, effort: small, ~1h)
   - Add `Object.keys(bundledLangs).includes(options.lang)` check before `loadLanguage`.
   - Add `Object.keys(bundledThemes).includes(theme)` check before `loadTheme`.
   - Throw a descriptive error or return a fallback instead of casting.

4. **Add explicit return types to all exported functions in `lib/api.ts`, `lib/orgUtils.ts`, `lib/shiki.ts`, and hooks** (P3, effort: small, ~1h)
   - `api.ts`: `Promise<Response>`.
   - `orgUtils.ts`: `string` and `'default' | 'secondary' | 'outline'`.
   - `shiki.ts`: `Promise<string>`.
   - `useOrganizations.ts`, `avatar/hooks.ts`: add explicit return object types.

5. **Add runtime validation for DiceBear schema in `avatar/hooks.ts`** (P2, effort: small, ~1h)
   - Replace `mod.schema as StyleSchema` with a `zod` or `io-ts` schema validation, or at least an `in` check before casting.
