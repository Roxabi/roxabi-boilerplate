# Type-Safety Audit — Partition P8 (Shared Packages)

**Scope:** `packages/ui`, `packages/types`, `packages/config`, `packages/email`, `packages/cli`, `packages/vitest-config`, `packages/playwright-config`
**Auditor:** Code Quality Auditor
**Date:** 2026-06-04

---

## Summary

P8 comprises **135 source files** (93 non-test, 42 test). The shared packages are **largely clean** of severe type-safety violations: zero `as any`, zero `as unknown as` in production code, zero `@ts-ignore`, zero non-null assertions (`!.`), and zero implicit `any` in exported signatures.

The **only P1 findings** are in `packages/cli` where runtime HTTP/JSON deserialization uses generic `as T` without validation. A **systemic P2 pattern** exists across `packages/ui` where **142 exported React component functions** lack explicit return type annotations. Test files contain a small number of justified `as unknown` and `@ts-expect-error` usages.

## Recommendations

1. **P1 — CLI deserialization:** Add `zod` or `valibot` schemas to validate API responses before casting. Replace `as T` with `schema.parse()` in `packages/cli/src/lib/client.ts`. Add a credential validator for `JSON.parse` in `packages/cli/src/lib/credentials.ts`.
2. **P2 — Component return types:** Add an explicit `: React.ReactNode` return type to all exported React component functions. This is a mechanical change that can be automated via a single codemod or a Biome/ESLint rule with auto-fix.
3. **P2 — CLI output casts:** Replace `as Record<string, unknown>` in `packages/cli/src/lib/output.ts` with a proper type guard or generic `Record<string, unknown>` parameter typing.
4. **P3 — Test casts:** Keep as-is. The `as unknown as MediaQueryList` and `@ts-expect-error` patterns in `useReducedMotion.test.ts` are standard for mocking browser APIs and testing edge cases. The `as ShadcnPreset` in `presets.test.ts` is acceptable for test assertions.
