### Summary
Partition P6 (web routes, integrations, hooks, config) has no explicit `any` annotations, `@ts-ignore`, `@ts-expect-error`, or non-null assertions (`!.`), but contains **9 P1** unsafe `as` casts on API responses and **13 P2** `res.json()` returns typed as `Promise<any>` inside generically-typed TanStack Query hooks, creating a runtime type-safety gap across the admin surface. Test files use `as unknown` to mock hook return types.

### Recommendations (prioritized, with effort estimate)

1. **Add runtime validation for API response bodies (P1, medium effort)**
   Replace `as` casts on `res.json()` with Zod or Valibot schemas for the 9 P1 locations (dashboard, login-handlers, account-reactivation, admin members, admin settings). This is the highest-impact fix because it closes a correctness and security gap for tenant-scoped data.

2. **Standardize API response parsing with a typed fetch wrapper (P2, medium effort)**
   Introduce a `fetchApi<T>` helper that calls `res.json()` and validates the result against a schema (e.g., Zod) before returning. Apply it to all `res.json()` calls in admin routes (13 P2 locations). This eliminates the `Promise<any>` runtime gap across the admin surface.

3. **Validate localStorage theme data before applying (P2, low effort)**
   In `theme-utils.ts:85`, replace the `JSON.parse` + `as` cast with a Zod schema for the stored shape, or at minimum add a `try/catch` with property validation. In `index.lazy.tsx`, keep the inline script as JS but add a `typeof` guard before accessing `config.radius` and `config.typography`.

4. **Remove redundant `as` casts (P3, trivial effort)**
   Remove `undefined as string | undefined` in `useCursorPagination.ts:22` and `language as string` in `CodeSnippet.tsx:30`. Both are already well-typed without the cast.

5. **Replace `as unknown` in tests with typed mock factories (P3, low effort)**
   In `members.test.tsx` and `settings.test.tsx`, create typed mock factories for `authClient.useActiveOrganization` and `useEnrichedSession` return values instead of `as unknown` casts.
