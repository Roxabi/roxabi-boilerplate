### Recommendations
1. **Add explicit return types to all public service methods** (UserService, UserPurgeService, OrganizationService, TenantService) — effort: 15 min. This surfaces the contract in the service layer and prevents silent signature drift when repository implementations change.
2. **Add explicit return types to all public controller methods** (UserController, OrganizationController) — effort: 10 min. Documents the HTTP response shape and makes the API contract visible at the entry point.
3. **Replace `as` casts in `TenantInterceptor` and `TenantService` with typed CLS wrappers** — effort: 10 min. Use `cls.get<string | null>('tenantId')` if `nestjs-cls` supports typed getters, or introduce a small typed helper to avoid `as` assertions on every CLS access.
4. **Eliminate `this.db!` in `TenantInterceptor.lookupOrganization`** — effort: 5 min. Refactor to pass `db` as a parameter from the caller (which already guards against null), or narrow the field type to `DrizzleDB` inside the method via an early return/assertion that the compiler understands. The `biome-ignore` comment documents the intent but does not remove the type-system bypass.
