### Summary
P6 (web routes, integrations, hooks) is clean of critical and high-severity issues. Two medium findings relate to `dangerouslySetInnerHTML` in the design-system page (latent XSS and CSS-injection surfaces), and two low findings cover missing `rel="noopener noreferrer"` and a placeholder GDPR contact. All RBAC routes are guarded with `enforceRoutePermission` or `requireAuth`/`requireGuest`. No hardcoded secrets, `eval`, `sql.raw`, or tenant-scoping leaks were found in the frontend code.

### Recommendations (prioritized, with effort estimate)
1. **Add `rel="noopener noreferrer"` to register links** (P3, 5 min) — Add `rel="noopener noreferrer"` to the `Link` components at `register.tsx:240` and `:245`.
2. **Normalize `credentials: 'include'` on all admin fetch calls** (P3, 15 min) — Audit all `fetch` calls under `routes/admin/` and add `{ credentials: 'include' }` where missing to ensure cookie-based auth is explicit and resilient to origin changes.
3. **Replace `dangerouslySetInnerHTML` in CodeSnippet with a safe rendering strategy** (P2, 2–4h) — Consider using `ref` + `innerHTML` only inside a `useEffect` after Shiki returns, or switch to a Shiki renderer that returns a React tree (e.g., `@shikijs/rehype` or `react-syntax-highlighter`). At minimum, add a runtime guard that rejects `code` strings containing `<` or `>` if the prop is not explicitly marked as safe.
4. **Sanitize localStorage-driven theme script** (P2, 1–2h) — In `design-system/index.lazy.tsx`, validate that parsed `localStorage` values conform to an expected schema before calling `style.setProperty`. Reject unexpected keys or values that contain `;` or `{` to close the CSS injection surface.
5. **Customize `legal.config.ts` before deployment** (P3, 5 min) — Replace `privacy@example.com` and other placeholder values with real company details.
