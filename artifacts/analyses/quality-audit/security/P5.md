### Summary
P5 partition (cross-cutting infra, DB, queue, v1 API) shows good security posture overall: no critical or high severity issues, proper tenant scoping via RLS and `TenantService.query()`, no SQL injection or unsafe eval patterns, and a robust auth guard chain (`AuthGuard` + `CustomThrottlerGuard`). Medium-severity findings are limited to TLS misconfiguration risk in Nodemailer and queue DB connection bypassing the RLS-enforced role. Low-severity findings are primarily log-level information leakage.

### Findings
| severity | file | line | description |
|---|------|------|-------------|
| P2 | apps/api/src/email/nodemailer.provider.ts | 21 | `ignoreTLS: !secure` disables TLS when `SMTP_SECURE` is false (default). In production, if `SMTP_HOST` is set without `SMTP_SECURE=true`, emails are sent in plain text, exposing email content and credentials. |
| P2 | apps/api/src/queue/queue.service.ts | 26 | `QueueService` uses `DATABASE_URL` for pg-boss connection, bypassing `DATABASE_APP_URL` (RLS-enforced role). If `DATABASE_URL` connects as a superuser, the queue worker bypasses RLS defense-in-depth. |
| P3 | apps/api/src/email/resend.provider.ts | 27-29 | Console fallback mode logs email text content and extracts URLs via regex, potentially leaking sensitive tokens (e.g., password reset links) in development logs. |
| P3 | apps/api/src/email/email.module.ts | 15 | When no `RESEND_API_KEY` or `SMTP_HOST` is configured, falls back to console-log provider which logs all email content including subjects and body text. |
| P3 | apps/api/src/throttler/filters/throttlerException.filter.ts | 52-56 | Logs rate limit `tracker` (IP address or user ID) and request path on 429 responses, potentially leaking user identities in application logs. |
| P3 | apps/api/src/common/filters/allExceptions.filter.ts | 92-98 | Logs full exception stack traces for 500 errors, potentially leaking internal implementation details (file paths, function names) in production logs. |

### Recommendations (prioritized, with effort estimate)

1. **Fix Nodemailer TLS default (P2)** — Change `ignoreTLS: !secure` to `ignoreTLS: false` and require explicit `SMTP_SECURE=true` for any non-localhost relay. Add a runtime warning if `SMTP_HOST` is set without `SMTP_SECURE=true` in non-development environments. (Effort: XS, 1 file)

2. **Queue service respects DATABASE_APP_URL (P2)** — Update `QueueService` to prefer `DATABASE_APP_URL` over `DATABASE_URL` for pg-boss, consistent with `DrizzleProvider`. If neither is set, warn and skip queue initialization. (Effort: XS, 1 file)

3. **Sanitize console email logging (P3)** — In `ResendEmailProvider` console fallback, redact URLs containing query parameters (e.g., `?token=...`) before logging. Consider logging only the recipient and subject, not the full text body. (Effort: XS, 1 file)

4. **Reduce log verbosity in exception filters (P3)** — In `AllExceptionsFilter`, log only the exception message (not the full stack trace) for 500 errors in production. Keep stack traces for development. In `ThrottlerExceptionFilter`, log only the tier name and correlation ID, not the tracker or path. (Effort: XS, 2 files)

5. **Add email provider configuration validation (P3)** — In `EmailModule` factory, throw an error in production if no email provider is configured (neither `RESEND_API_KEY` nor `SMTP_HOST`), instead of silently falling back to console logging. (Effort: XS, 1 file)
