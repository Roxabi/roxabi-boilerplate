### Recommendations (prioritized, with effort estimate)
1. **Break consent circular dependency** (P1) — 1h
   - Move `ConsentBanner` and `ConsentModal` out of `lib/consent/consentProvider.tsx` imports. Use a `ConsentUI` wrapper in `components/` or composition pattern so `lib/consent/` only exports the context/provider/hooks, and `components/consent/` composes the UI.
2. **Extract admin API layer** (P2) — 2-3h
   - Create `lib/admin/api.ts` (or `lib/admin/users.ts`, `lib/admin/orgs.ts`, `lib/admin/invitations.ts`, `lib/admin/featureFlags.ts`) and move all inline `fetch` calls, `useMutation`, and `useQuery` hooks from `components/admin/*` into these modules. Components should only import and call these hooks.
3. **Deduplicate admin components** (P2) — 1-2h
   - Remove the local `BanDialog` from `UserActions.tsx` and import the shared one from `components/admin/BanDialog.tsx`.
   - Extract `ImpactSummary` to `components/admin/ImpactSummary.tsx` and share it between `OrgActions.tsx` and `OrgListContextMenu.tsx`.
   - Extract `useOrgMutations` and `useUserMutations` to `lib/admin/` and share across context menus and action buttons.
4. **Consolidate profile API calls** (P2) — 30m
   - Merge `fetchUserProfile` (`lib/api.ts`) and `getProfile` (`lib/profile.ts`) into a single function in `lib/profile.ts` with consistent error handling. Update `lib/useOrganizations.ts` to use the consolidated helper if needed.
5. **Move constants out of `@repo/types`** (P3) — 1h
   - Move `SENSITIVE_FIELDS`, `AVATAR_STYLES`, `DICEBEAR_CDN_BASE` to `@repo/config` or a dedicated `@repo/constants` package. Update all imports.
6. **Introduce client-side API client wrapper** (P3) — 1-2h
   - Create a lightweight `fetch`-based client in `lib/apiClient.ts` (or extend `lib/api.ts`) with uniform error handling, JSON parsing, and credentials. Align it with the server-side `ofetch` client in terms of behavior (e.g., correlation ID, retry). Migrate all raw `fetch` calls in components and `lib/api.ts` to use this wrapper.
7. **Move domain utilities to `lib/`** (P3) — 30m
   - Move `validateKey`, `slugify`, `buildTree`, and `coerceValue` to their respective `lib/` modules (e.g., `lib/featureFlags/validation.ts`, `lib/admin/tree.ts`).
8. **Use `lib/formatDate.ts` consistently** (P3) — 15m
   - Replace inline `new Date(...).toLocaleDateString()` calls in `PendingInvitations.tsx` and `MembersTable.tsx` with `formatDate` or `formatTimestamp` from `lib/formatDate.ts`.
