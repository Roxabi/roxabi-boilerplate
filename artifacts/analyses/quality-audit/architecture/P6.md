# P6 Architecture Quality Audit — Web Routes, Integrations, Hooks

## Summary

Partition P6 (apps/web routes, hooks, integrations, config) contains several business-logic leaks and inconsistent state-management patterns. The most critical issue is a raw `useEffect` + `fetch` anti-pattern in the API-keys module that bypasses the project's TanStack Query standard. No `@repo/types/api` boundary violations, direct DB access, or deep relative imports were found.

## Recommendations

### 1. Migrate `useApiKeys` to TanStack Query (High effort: 1-2h)
Replace the raw `useEffect` + `fetch` in `settings/api-keys/-hooks.ts` with `useQuery` and `useMutation`. Use `queryKey` factory from `lib/admin/queryKeys.ts` for consistency. Reference `useMembersPageQueries` in `admin/members.tsx` as a model.

### 2. Extract `fetchOwnedOrgsForUser` to API layer (High effort: 1-2h)
Move the org-ownership resolution logic to `apps/api` (e.g., `GET /api/account/owned-orgs`) or to a dedicated service under `lib/account.ts`. The route file should only call the API and render UI.

### 3. Deduplicate `slugify` (Low effort: 15min)
Move `slugify` to `lib/string.ts` (or `lib/slug.ts`) and import it in both `admin/settings.tsx` and `admin/-organizations-create-dialog.tsx`. Remove the duplicate.

### 4. Centralize audit action constants (Low effort: 30min)
Either add a `GET /api/admin/audit-log-actions` endpoint or move `AUDIT_ACTION_OPTIONS` to a shared constants file (e.g., `lib/admin/constants.ts`).

### 5. Extract module-level mutation hooks to `lib/admin/` (Medium effort: 2-3h)
Move `useEditUserMutation`, `useOrgEditMutation`, and `useCreateOrgMutation` to `lib/admin/mutations.ts` (or per-domain files). This aligns with the `admin/feature-flags.tsx` pattern which already keeps mutations in the route file but uses `useMutation` properly. Actually, `admin/feature-flags.tsx` has `useToggleFeatureFlag` and `useDeleteFeatureFlag` inside the route file — these are small. The larger ones should be extracted.

### 6. Extract route-specific business logic to `hooks/` or `lib/` (Medium effort: 3-4h)
Move `handleRegister` to `lib/auth/register.ts`, `useAccountDeletionCheck`/`useAutoSelectOrg` to `hooks/useDashboard.ts`, `parseProfileResponse`/`useProfileData` to `hooks/useProfileData.ts`, and `groupPermissionsByResource` to `lib/permissions.ts`.

### 7. Inline mutations in `system-settings.tsx` (Low effort: 30min)
Wrap `handleSave` in `useMutation` for consistency and to leverage `isPending` state instead of manual loading.

### 8. Move API helpers to `lib/admin/api.ts` (Low effort: 1h)
Move `fetchDeletionImpact`, `deleteOrganization`, `fetchMembers`, and `fetchRoles` to a centralized API module.

### 9. Move validation constants to shared config (Low effort: 30min)
Move `SLUG_REGEX`, `EMAIL_REGEX`, and `ROLE_OPTIONS` to `lib/validation.ts` or `lib/constants.ts`.

### 10. Consider route-file splitting (Medium effort: 4-6h)
The following route files are >300 lines and contain multiple sub-components, hooks, and helpers:
- `admin/settings.tsx` (~500 lines)
- `admin/members.tsx` (~350 lines)
- `admin/users.$userId.tsx` (~590 lines)
- `admin/organizations.$orgId.tsx` (~500 lines)
- `register.tsx` (~410 lines)
- `settings/profile.lazy.tsx` (~375 lines)
- `settings/-account-delete.tsx` (~355 lines)

Extract sub-components to `components/admin/` or `components/settings/` and extract hooks to `hooks/` or `lib/`. This improves testability and readability.

## Positive Observations

- **Permission guards**: `enforceRoutePermission`, `requireAuth`, `requireGuest` are used consistently and correctly.
- **No boundary violations**: Zero `@repo/types/api` imports found in P6.
- **No DB access**: No direct database/repository imports in frontend files.
- **No deep imports**: No `../../../` (4+ level) relative imports found.
- **Good abstractions**: `useCursorPagination` is a clean, reusable hook. `admin/feature-flags.tsx` correctly uses `useMutation` for mutations.
- **SSR cache priming**: `settings/api-keys/index.tsx` correctly uses `ensureQueryData` to prime the React Query cache with the session, avoiding an extra network request.
