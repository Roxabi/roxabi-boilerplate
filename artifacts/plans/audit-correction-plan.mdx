**Rationale:**
- Phase 1 changes DB schema (`members.deletedAt`, `users.tenantId`, RLS enable). All subsequent phases reference these schema changes (e.g., `whereActive(members)`, `tenantId` filters). Schema must be stable before downstream work begins.
- Phases 2‚Äì5 are code-only (no schema changes) and touch largely independent file groups. After Phase 1 completes, they can run in parallel.
- **Recommended:** Phase 1 alone, then Phase 2‚Äì5 in parallel, with a reconcile step before final verification.
